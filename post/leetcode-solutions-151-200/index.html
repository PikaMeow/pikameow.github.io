155. Min Stack

easy 2019-08-26

思路一：使用2个栈，一个栈保存输入的数据，一个栈保存到当前为止栈中数据的最小值。

class MinStack {
  private:
    stack<int> s, ms;
  public:
    void push(int x) {
        s.push(x);
        (ms.empty() || x < ms.top()) ? ms.push(x) : ms.push(ms.top()); // 保存当前栈中最小值
    }
    void pop() { s.pop(); ms.pop(); }
    int top() { return s.top(); }
    int getMin() { return ms.top(); }
};

思路二：使用1个栈，并保存最小值min。在数据入栈时，如果比min还小，那么就先将min入栈，再更新min值；弹出数据时，如果发现当前数据与min相同，那么先弹出后，更新min为当前栈顶元素，然后再弹出一个数据。

思路三：使用1个栈，并保存最小值min。数据x入栈时存储差值x-min，并更新min；弹出时如果发现为负，说明弹出了最小值，更新min；取栈顶元素时，根据正负选择返回 min 或者 min + top。 PS: 这种方法需要用long类型的栈，因为差值可能超过int类型的存储范围。

160. Intersection of Two Linked Lists

easy 2019-08-26

思路一

Step1: 从 headA 和 headB 开始分别遍历链表，得到各自的长度 lenA 和 lenB。

Step2: 从较长的链表头开始，移动 abs(lenA - lenB) 步，使得两个链表等长。

Step3: 同时遍历两个链表，指针相同时的位置就是交点。

思路二

两个指针 pA 和 pB 分别从两个链表头向后移动，pA移动到末尾后指向 headB，pB移动到末尾后指向 headA，接下来两者一定会相交，如果是NULL就说明两链表无交点。 一个详细的图示

ListNode *pA = headA, *pB = headB;
if (!pA || !pB) return NULL;
while (pA != pB) {
    pA = pA ? pA->next : headB;
    pB = pB ? pB->next : headA;
}
return pA;

167. Two Sum II - Input array is sorted

easy 2019-07-09

由于数组已经排好序了，首先我们可以使用第1题的方法，从左至右取元素，然后对余下的元素进行二分搜索，复杂度为O(lg⁡1+⋯+lg⁡n)=O(n2lg⁡n)O(\lg1+\cdots+\lg n)=O(\frac{n}{2}\lg n)O(lg1+⋯+lgn)=O(2n​lgn)。但我们发现有线性的方法，使用首尾2个指针，判断其和与 target 的大小，如果比它大，那么将首指针后移；如果比它小，那么将尾指针前移。由于这样的元素总是存在，所以一定能够找到。

int beg = 0, end = numbers.size() - 1;
while (beg < end) {
    int sum = numbers[beg] + numbers[end];
    if (sum == target) return {beg + 1, end + 1};
    else if (sum > target) end--;
    else beg++;
}

169. Majority Element

easy 2019-07-08

这个问题的解法很多，参考了花花酱 LeetCode 169，给出了5类，共8种解法，列表如下：

首先是基于搜索的2种方法，我们可以使用Hash Table(即<unordered_map>)或者是BST(即<map>)来通过遍历一遍数组，将每个元素出现的次数统计下来，一旦发现有元素出现超过 n/2 次，则返回。

unordered_map<int, int> table; // hash table
map<int, int> table; // BST
for (auto i : nums) {
    if (table.find(i) != table.end()) table[i]++;
    else table[i] = 1;
    if (table[i] > nums.size() / 2) return i;
}

由于数组中必然存在出现次数超过 n/2 次的元素，因此我们可以随机地选取一个元素，它会有至少1/2的概率是我们需要的结果，如果不是，那么就再取一个，直到满足条件为止。

srand(time(NULL));
int len = nums.size();
while (true) {
    int pivot = nums[rand() % len], count = 0;
    for (auto num : nums) if (num == pivot && ++count > len / 2) return pivot;
  	//这里的if语句巧妙利用了短路特性，只有 num == pivot 时，count 才会增加，并且先增加后判断
}

之后是基于Vote的方法，先通过一个例子说明：

假设我们有数组[1,2,1,2,1]，那么主要元素为1，我们从左向右看，可以视为有两组1,2抵消了，最后只剩下一个1，那么1就是主要元素。

基于这样的思想，我们从左至右遍历数组，并保存2个值 candidate 和 count, 表示当前候选的元素以及出现的次数。之后如果元素值与 candidate 不同，则减小 count, 如果 count 减小至0，那么就说明该 candidate 被抵消完了，那么把前面的这些元素都除去，剩下的数组中的主要元素就是整个数组的主要元素。这个方法叫做 Boyer-Moore Vote。

int cand = 0, count = 0;
for (auto i : nums) {
    if (count == 0) { cand = i; count++; } 
  	else {
      	if (cand == i) count++;
      	else count--;
    }
}
return cand;

与这种思想类似，我们可以对每一个比特位都进行这样的操作，来确定每个比特位的“主要元素”。

int major = 0, half = nums.size() / 2;
for (int i = 0; i < 32; i++) {
    int mask = 1 << i, count = 0;
    for (auto num : nums) if (num & mask) count++;
    if (count > half) major |= mask;
}
return major;

然后是基于排序的算法，我们发现，如果有一个元素出现次数超过 n/2，那么排完序之后，处于中间位置的元素的值一定是该元素，返回即可。另外，由于我们只需要顺序上处在中间位置的元素，因此不需要完全排序，只需要部分排序，保证在该元素之前的元素都更小，该元素之后的元素都更大。实现的方法与快速排序的Partition过程类似。

nth_element(nums.begin(), nums.begin() + nums.size() / 2, nums.end());
// 找到区间 [ nums.begin(), nums.end() ) 内，升序排列后处于数组正中间的元素的值
return nums[nums.size() / 2];

最后，我们还可以通过递归的方式来做，每次将数组划分为两半，然后返回各自的主要元素及出现的次数。最后合并时，如果两个子数组的主要元素相同，那么返回该元素及在两个子数组中出现次数之和；如果不同，那么返回出现次数较多的元素，并计算它在另一个子数组中出现的次数。

int majorityElement(vector<int> &nums) { 
  	return majorityElement(nums, 0, nums.size() - 1).first;
}
pair<int, int> majorityElement(vector<int> &nums, int beg, int end) {
    if (beg == end) return {nums[beg], 1}; // 可以直接用列表的形式返回pair
    int mid = (beg + end) >> 1;
    auto ml = majorityElement(nums, beg, mid);
    auto mr = majorityElement(nums, mid + 1, end);
    if (ml.first == mr.first) return {ml.first, ml.second + mr.second};
    if (ml.second > mr.second) return {ml.first, ml.second + count(nums.begin() + mid + 1, nums.begin() + end + 1, ml.first)};
    // count(a,b,c) 函数返回在区间 [a,b) 范围内值等于c的元素的个数
    else return {mr.first, mr.second + count(nums.begin() + beg, nums.begin() + mid + 1, mr.first)};
}

172. Factorial Trailing Zeroes

easy 2019-08-25

n!n!n!末尾的0是从哪里来的呢？每多乘一个10，数字末尾就多了一个0，而对10进行质因数分解：10=2×5，也就是说0的数目取决于2的个数和5的个数中的较小值。对于阶乘来说，每隔一个数就有一个偶数，因此5的数目要少一些。

怎么计算5的数目呢？n!n!n!中5的个数为n5+n52+n53+⋯\frac{n}{5}+\frac{n}{5^2}+\frac{n}{5^3}+\cdots5n​+52n​+53n​+⋯ 即每个5的倍数贡献1个5，每个25的倍数再多贡献1个5，...，直到5的幂超过nnn。

for (long long i = 5; n / i; i *= 5) { count += n / i; }

189. Rotate Array

easy 2019-07-03

首先处理边界情况：

  1. k = k % length，当k超过数组长度时，可以减小k，而达到同样的效果。
  2. if(!k || length <= 1) return; 如果发现不用移动，或者数组长度不超过1，那么直接返回

方法一：使用额外数组

新建立一个数组，然后将原数组各个元素移动后应该放置的位置直接放在新数组中。

vector<int> ans(len);
for (int i = 0; i < len; i++)  ans[(i + k) % len] = nums[i];
nums = ans;

方法二：多次反转数组

通过观察我们发现，进行旋转的结果与多次将数组反转的结果是相同的。例如：[1,2,3,4,5], k=2

  1. 反转整个数组：[5,4,3,2,1]
  2. 再反转前k个元素：[4,5,3,2,1]
  3. 再反转后n-k个元素：[4,5,1,2,3]

这与向右移动k个元素的结果是相同的。

void rotate(vector<int> &nums, int k) {
    reverse( nums.begin(), nums.end() );
    reverse( nums.begin(), nums.begin() + k );
    reverse( nums.begin() + k, nums.end() );
}

方法三：原地移动

使用一个临时变量tmp，将各元素移动后所在的位置直接放到原数组中。

通过观察我们发现，在将元素移位时，有可能一次就将所有的元素移到正确位置上，也有可能形成若干个环，而环的个数恰巧是gcd(length, k)。因此我们只需要将所有的环都考虑到即可。移位的时候我们发现一个有趣的现象：

以数组[1,2,3], k=1为例，数组的移动分为2种：左移 (结果为[2,3,1]) 和右移 (结果为[3,1,2])。

如果采取连锁交换的方式，即下标i与下标i+k的元素交换，从i=0依次向右，那么会得到左移的结果；如果将下标i与下标i-k的元素交换，从i=len-1依次向左，那么会得到右移的结果。

而如果采取固定位置与其它位置交换的方式，如果从i=0开始，将其依次与i+k,i+2k位置的元素交换，那么会得到右移的结果；如果从i=len-1开始，依次与i-k,i-2k位置的元素交换，又会得到左移的结果。

所以出于数组在内存中存放的连续性和读取方式的考虑，如果需要左移，可采取上一种方式，如果需要右移，可采取下一种方式，效率更高。

for (int i = 0; i < gcd(len, k); i++) {
    for (int j = i + k, p = j; j != i; j = p) {  
      tmp = nums[i];  nums[i] = nums[p];  nums[p] = tmp; // 将第i个与第p个元素交换
      p = (j + k) % len;
    }
}

190. Reverse Bits

easy 2019-07-04

之前在CSAPP上就遇到过这个题，又忘记解法了。首先是比较naive的解法：新声明一个数，然后将这个数的二进制位倒序存入新的数中。

for (int i = 0; i < 32; i++)  if (n & (uint32_t)(1 << i))  ans |= (1 << (31 - i));

但是如果限制不允许使用for和if这样的条件控制语句，那么仅靠位运算是否可行呢？我们可以将32位使用递归的思想来处理，即首先将前一半和后一半互换，然后对每一块再互换两半，直到块的长度为1。这可以通过使用合适的掩码和移位操作来实现，所计算的耗时是O(log⁡232)=O(5)O(\log_232)=O(5)O(log2​32)=O(5)。

n = ((n & 0xffff0000) >> 16) | ((n & 0x0000ffff) << 16);
n = ((n & 0xff00ff00) >> 8) | ((n & 0x00ff00ff) << 8);
n = ((n & 0xf0f0f0f0) >> 4) | ((n & 0x0f0f0f0f) << 4);
n = ((n & 0xcccccccc) >> 2) | ((n & 0x33333333) << 2);
n = ((n & 0xaaaaaaaa) >> 1) | ((n & 0x55555555) << 1);
return n;

191. Number of 1 Bits

easy 2019-07-05

一个讲解很棒的链接：上, 下

首先排除一个一个数的思路。可以使用和上一题类似的递归的思想，先计算相邻2位的1的数目(即加和)，再计算相邻4位的1的数目，直到计算完所有32位。这种方法叫做 Shift and Add.

n = (n & 0x55555555) + ((n >> 1) & 0x55555555);
n = (n & 0x33333333) + ((n >> 2) & 0x33333333);
n = (n & 0x0f0f0f0f) + ((n >> 4) & 0x0f0f0f0f);
n = (n & 0x00ff00ff) + ((n >> 8) & 0x00ff00ff);
n = (n & 0x0000ffff) + ((n >> 16) & 0x0000ffff);
return n;

通过分析可以发现，这种方法其中的一些运算是多余的，我们可以做一些优化。我们的发现是2个k位的最大和(2k)是否超过了k位二进制数表示的范围。对于前2步都是为否，所以要正常地通过掩码先与后加。[ 还发现对于2个1位的加和a+b=(2*a+b)-a=(ab)-a ]，所以第1步可以使用减法运算省掉一次与运算。

对于第3步，2个4位数的和最大为8，可以用4位二进制表示，因此可以先加，但是由于2个8位数的和最大为16，超出了4位二进制的范围，所以仍然要和掩码进行与运算，否则可能影响后面的计算。

对于第4,5步，即不会出现和超出范围，也不会出现“污染”后面计算的情况，所以不需要再进行与运算。

最后，由于1的个数最多为32，可以用6位二进制表示，所以最后只取计算的最后6位结果即可。

n -= ((n >> 1) & 0x55555555);
n = (n & 0x33333333) + ((n >> 2) & 0x33333333);
n = (n + (n >> 4)) & 0x0f0f0f0f;
n += (n >> 8);
n += (n >> 16);
return n & 0x3f;

另外我们还有一个观察，在第3步执行完后，数字被分成了4个长度为8的块，如果可以使用乘法运算，那么可以一步计算完成(列竖式计算很容易发现)，然后再将后面的24位去掉，只取前8位就是我们需要的结果。即最后3行可以改为：

return (n * 0x01010101) >> 24;

除了这种方法之外，还有一种运算更少的方法。算法以汇编的形式写在了备忘录HEKMEM的第169个算法，因此又称HEKMEM169。

算法首先使用了3个1位相加的策略。对于相邻的3个数(abc)，它表示的数字大小为4a+2b+c，因此先右移1位并和掩码与运算后得到(ab),即2a+b，再右移2位得到(a)，用原数相减得 a+b+c。

然后使用2个3位相加的策略，得到每6位自成一组，对于任意一个相邻的6位，假设是(ab)，那么它表示的数字大小为64a+b，而它与a+b模63同余。所以对于整个数，都可以看做是递归使用了若干个(ab)，由于1的个数的范围是[0,32]，那么它一定为 n%63。

n -= ((n >> 1) & 033333333333) + ((n >> 2) & 011111111111); // 0开头表示八进制
n = (n + (n >> 3)) & 030707070707;
return n % 63;
  • 注：既然关于模63同余是可行的，那么使用之前的方法，也可以出现模15和模255同余的情况，是否也可行呢？依然是考虑到1的个数在[0,32]之间，模16肯定不可行，而模255是可行的，也就是将算法最后3行改为return n % 255;即可。
