2. Add Two Numbers

medium 2019-07-14

给定2个用链表存储各位的数，将其相加并返回用链表表示的和。

可以直接使用 digit-by-digit 的加法，只需要注意进位和链表节点为空的情况，同时可以使用递归或非递归的方式来写：

ListNode *addTwoNumbers(ListNode *l1, ListNode *l2) {
    if (!l1 && !l2) return NULL;
    else if (!l1) return l2;
    else if (!l2) return l1;
    int a       = l1->val + l2->val;
    ListNode *p = new ListNode(a % 10); // 存下当前位置应该存储的数字
    p->next     = addTwoNumbers(l1->next, l2->next);
    if (a >= 10) p->next = addTwoNumbers(p->next, new ListNode(1)); // 处理进位
    return p;
}

但是这种情况相当于重新建立了一个新的链表，空间效率比较低，因此我们考虑使用原链表来保存和。需要注意当某个数位数较多(链表长度大)时，需要将该链表多余的位链接到结果的链表的后面。

ListNode *head = l1;
while (l1->next && l2->next) { // 处理相加及进位
    l1->val = l1->val + l2->val;
    if (l1->val >= 10) { l1->next->val++; l1->val -= 10; }
    l1 = l1->next; l2 = l2->next;
} 
if (l2->next) l1->next = l2->next; // 如果l2比较长，那么把剩余部分链到l1后
l1->val = l1->val + l2->val;
while (l1 && l1->val >= 10) { // 处理可能存在的连续进位的情况 (eg:999+1)
    if (l1->next) l1->next->val++;
    else ListNode *p = new ListNode(1); l1->next = p;
    l1->val -= 10; l1 = l1->next;
}
return head;

3. Longest Substring Without Repeating Characters

medium 2019-07-15

滑动窗口法(双指针法)。我们使用2个指针i和j，以及一个hash表用来保存某个字符是否被访问过。我们用i来遍历字符串，并更改hash表中每个字符的状态。如果出现重复，那么保存当前子串长度，并移动j直到s[i]==s[j]，并更改其间字符在hash表中的状态，随后从j+1的位置作为新的子串的开始。最后返回这些子串长度的最大值。

这种方法在最坏情况下，我们需要将i和j都从0移动到len(s)，复杂度为O(2n)O(2n)O(2n)。我们又发现，hash表不仅可以保存字符是否访问过的信息，还可以保存字符的位置。因此我们在遇到重复字符的时候，可以直接将hash表中该字符位置（即上一次出现该字符的位置）作为新的子串的起点。这样只需要遍历一遍数组即可，复杂度为O(n)O(n)O(n)。

vector<int> visited(128, -1);
int len = 0, start = -1; // start用来表示当前子串起始位置的前一个位置
for (int i = 0; i < s.size(); i++) {
    if (visited[s[i]] > start) start = visited[s[i]];
  	// 如果条件满足，说明出现了重复元素，那么更新 start
    visited[s[i]] = i;
    len = max(len, i - start); // i-start 表示当前子串的长度
}
return len;

5. Longest Palindromic Substring

medium 2019-07-17

DP方法。在判定某个子串是否是回文子串时，我们可以避免一些不必要的比较，例如对于abwba，如果已知bwb是回文子串，那么在判断abwba时就不需要从头比较。设dp[i][j]表示子串sijs_{ij}sij​是否为回文子串，那么它满足递推公式：

dp[i][j]={dp[i+1][j−1]if s[i]=s[j]falseo.w.dp[i][j]= \begin{cases} dp[i+1][j-1]& if\ s[i]=s[j]\\ false& o.w. \end{cases}dp[i][j]={dp[i+1][j−1]false​if s[i]=s[j]o.w.​

基础情况是dp[i][i]=true,dp[i][i+1]=(s[i]==s[i+1])dp[i][i]=true, dp[i][i+1]=(s[i]==s[i+1])dp[i][i]=true,dp[i][i+1]=(s[i]==s[i+1])。根据递推公式，我们可以实现如下：

const int n = s.size();
if (!n) return s; // 处理空串情况
bool dp[n][n];
int start = 0, len = 1;
fill_n(&dp[0][0], n *n, false); // fill(a,b,c) 将从地址a开始的b个元素赋值为c
for (int i = 0; i < n; i++) { // i表示当前判断的字符串的末尾
    dp[i][i] = true;
    for (int j = 0; j < i; j++) { // j表示当前判断的字符串的开头
        dp[j][i] = (s[j] == s[i]) && (i - j < 2 || dp[j + 1][i - 1]);
        // 若i-j<2，那么只需要看s[i]==s[j]，否则可以根据之前的结果来判断
        if (dp[j][i] && i - j + 1 > len) { start = j; len   = i - j + 1; }
    }
}
return s.substr(start, len);

这种方法的时间和空间复杂度都为O(n2)O(n^2)O(n2)。

我们还有另一种思路，即以每个字符即字符间隙作为中心字符，然后向两边扩展判断回文子串的长度，最后取最长的回文子串即可。时间复杂度仍然为O(n2)O(n^2)O(n2)，但是空间复杂度为O(1)O(1)O(1)。

int n = s.size(), start = 0, len = 0;
for (int i = 0; i < n; ) {
    int beg = i, end = i; // 子串的首和尾
    while (end < n - 1 && s[end] == s[end + 1]) end++; // 处理子串中心在间隙的情况
    i = end + 1; // 如果是字符为中心，那么正常自增；如果是间隙为中心，那么可以跨越那些重复的字符
    while (beg > 0 && end < n - 1 && s[beg - 1] == s[end + 1]) { beg--; end++; } // 中心扩展
    if (end - beg + 1 > len) { start = beg; len = end - beg + 1; }
}
return s.substr(start, len);

最后介绍一种时间何和空间复杂度都为O(n)O(n)O(n)的Manacher算法，参考了链接的讲解。算法的思想是尽可能利用已经确定的回文子串的信息。

例如对于字符串abawabaf，我们从左至右，将每个字符作为中心向两端扩展，来确定以该字符为中心的回文子串的长度。假设我们知道以第[1,2,3,4]个字符为中心的回文子串长度分别为[1,3,1,7]，那么就可以知道第1-3个字符和第5-7个字符是对称的。

如果我们想知道以第5个字符为中心的回文子串长度，已知它与第3个字符关于第4个字符对称，那么因为以第3个字符为中心的回文子串长度为1，可得第2和第4个字符不同；又由以第4个字符为中心的回文子串长度为7，可得第2和第6个字符相同，因此第4和第6个字符不同，那么以第5个字符为中心的回文子串长度也为1。

如果我们想知道以第6个字符为中心的回文子串长度，已知它与第2个字符关于第4个字符对称，而且以第4个字符为中心的回文子串长度为7，可得第1-3个字符与第5-7个字符对称；又知以第2个字符为中心的回文子串长度为3，那么以第6个字符为中心的回文子串也至少为3。为什么是至少呢？因此第7个字符已经是我们所知道的信息的最右边界，尚不清楚第8个字符与第4个字符是否相同，因此需要以第6个字符为中心继续向边界扩展来判断。

通过这个例子我们可以发现，利用之前位置的回文子串的信息来判断的基础是当前已经确定的回文子串的最右边界。以某个字符为中心，如果在边界外，那么没有信息可以利用，只能进行最基础的中心扩展来确定，同时也是在扩大最右边界；如果在边界内，则又分2种情况：如果能够确定在到达边界处时已经不是回文子串，就可以直接确定结果；如果在到达边界处时还是回文子串，那么就要以此为基础继续扩展，来确定最终长度。

还有一个细节问题，字符个数可能为奇或偶，因此我们在字符串的首尾和各字符之间都加一个相同的且不在字符串中的字符#，就可以使回文串的中心总是一个固定的字符，并且保持与原字符串的回文属性不变。

string es = "#";
for (auto c : s) { es += c; es += '#'; }
int len = es.size();
int r = 0, c = 0; // 保存当前的 最右边界 和 中心字符
vector<int> ans(len, 0);
for (int i = 0; i < len; i++) {
    ans[i]  = r <= i ? 0 : min(r - i, ans[2 * c - i]);
    // 如果 r<=i，说明当前字符已经在边界处，需要进行中心扩展；如果r>i，那么如果 ans[2*c-i]<r-i，说明到边界时已经不是回文子串，否则还需要继续进行扩展
    int beg = i - ans[i], end = i + ans[i];
    while (beg > 0 && end + 1 < len && es[beg - 1] == es[end + 1]) {
        ans[i]++; beg--; end++;
    }
    if (i + ans[i] > r) { r = i + ans[i]; c = i; } // 扩展最右边界，以及对应的中心字符
}
c = max_element(ans.begin(), ans.end()) - ans.begin();
// max_element(a,b)返回迭代器区间[a,b)中指向最大元素的迭代器，减去a可以得到最大元素的索引
return s.substr((c - ans[c]) / 2, ans[c]);
// 对应到原字符中的位置

6. ZigZag Conversion

medium 2017-07-17

我们计算出在按照W型排列之后，各个字符应该位于哪一行。我们定义一个cycle为W型中一个V型的长度，len(cycle)=2*numRows-2。那么对于第iii行，分3种情况，令k∈Nk\in\mathbb{N}k∈N：

  1. 如果i=0i=0i=0或i=numRows−1i=numRows-1i=numRows−1，那么位于该行的字符索引为k∗CycleLen+ik*CycleLen+ik∗CycleLen+i
  2. 其余情况，位于第iii行的字符索引为k∗CycleLen+ik*CycleLen+ik∗CycleLen+i和(k+1)∗CycleLen−i(k+1)*CycleLen-i(k+1)∗CycleLen−i
if (numRows == 1) return s;
int len = s.size(), cycle = 2 * numRows - 2;
string ans = "";
for (int i = 0; i < numRows; i++) {
    for (int j = 0; i + j < len; j += cycle) { // j 表示 k * CycleLen
        ans += s[i + j];
        if (i != 0 && i != numRows - 1 && j + cycle - i < len)
            ans += s[j + cycle - i];
    }
}
return ans;

7. Reverse Integer

easy 2019-02-01

int tmp, y = 0;
while (x) {
    tmp = y * 10 + x % 10;
    if ((x > 0 && (tmp / 10 < y)) || (x < 0 && (tmp / 10 > y)))
        return 0; // 根据x的正负判断是否溢出
    y = tmp;
    x /= 10;
}
return y;

9. Palindrome Number

easy 2019-02-01

一个直接的思路是按照第7题将数字反转过来然后判断是否相等。另外：

  1. 将数字的最高位和最低位进行比较，然后是次高位和次低位，以此类推

    if (x < 0) return false;
int base = 1, tmp = x / 10;
while (tmp) { base *= 10; tmp /= 10; }
while (base > 1) {
    int last = x % 10, first = x / base; // 提取最低位和最高位的数字
    if (last != first) return false;
    x = (x / 10) % (base / 10); // 将x除去最高位和最低位
    base /= 100;
}
return true;

  2. 将数字的后半部分反转，然后与前半部分比较

    if (x < 0 || (x != 0 && x % 10 == 0)) return false;
int y = 0;
while (x > y) {
    y = 10 * y + x % 10; // 将x的后一半进行反转，存储在y中
    x /= 10;             // 将x的后一半去除
}
return (x == y || x == y / 10);
