141. Linked List Cycle

easy 2019-07-02

方法一：两个指针追赶法

使用2个指针p和q指向链表头，然后p每次前进2步，q每次前进1步，如果p到达链表尾，那么说明没有环；每次前进后判断(p==q || p==q.next)，如果为真，表示p通过环追赶上了q，说明有环。

时间效率分析：假设长度为nnn的链表有环，其中无环部分长度为aaa，有环部分为bbb。对指针qqq来说，到进入环的时刻，耗时为O(a)O(a)O(a)，此时ppp已经在环中。然后变成指针的追赶问题，两者的最大距离为bbb，而速度差为111，因此ppp追上qqq耗时最多为O(b)O(b)O(b)，加起来为O(n)O(n)O(n)。

ListNode *p = head->next, *q = head;
while (p) {
    p = p->next;  q = q->next;
    if (!p || !(p->next))  break;
    p = p->next;
    if (p == q || p == q->next)  return true; // 仅用 p==q 也可，但可能会更慢
}
return false;

方法二：倒转链表法

首先将链表倒转过来，如果链表中有环，那么倒转之后的rhead指针与原来的head指针相同。需要注意的是，判断是要保证链表元素大于1(含循环)。

ListNode *reverse(ListNode *head) {
    ListNode *p = NULL, *q = NULL; // prev and follow pointer
    while (head) {
        q          = head->next; // 找到下一个元素
        head->next = p; // 将当前head的next反向
        p          = head; // 将当前head置为prev
        head       = q; // 移动至下一个元素
    }
    return p;
}

方法三：哈希映射法

使用数据结构unordered_map，即用哈希表来保存链表的每个指针。然后遍历列表，如果发现键已经存在表中，说明链表有环。

unordered_map<ListNode *, int> table; // 键有用，值无用，可以随意设置
while (head) {
    table.insert(make_pair(head, head->val));
    head = head->next;
    if (table.find(head) != table.end())  return true;
}
return false;
