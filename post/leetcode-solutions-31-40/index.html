31. Next Permutation

medium 2019-07-21

使用组合数学中生成字典序的方法：求(p)=p1⋯pn(p)=p_1\cdots p_n(p)=p1​⋯pn​的下一个排列

  1. 求i=max⁡{j∣pj−1<pj}i=\max\{j|p_{j-1}<p_j\}i=max{j∣pj−1​<pj​}，即最后一个正序
  2. 求j=max⁡{k∣pi−1<pk}j=\max\{k|p_{i-1}<p_k\}j=max{k∣pi−1​<pk​}，即最后一个大于pi−1p_{i-1}pi−1​的数
  3. 互换pi−1p_{i-1}pi−1​与pjp_jpj​，然后反排pjp_jpj​后面的数

除此之外，只要注意当已经是最大序时，需要反转成最小的字典序即可。

int len = nums.size(), i = 0, j, k;
for (j = len - 1; j >= 1; j--) {
    if (nums[j - 1] < nums[j]) { i = j; break; }
}
if (i != 0) { // i=0时说明当前排列已经是最大字典序
    for (k = len - 1; k >= i; k--) {
        if (nums[i - 1] < nums[k]) { j = k; break; }
    }
    swap(nums[i - 1], nums[j]);
}
reverse(nums.begin() + i, nums.end()); // 反转区间[i,end)之间的元素

33. Search in Rotated Sorted Array

medium 2019-07-23

为了能够运用二分查找的方法，我们需要先找到数组是在哪里被旋转的。考虑到数组中没有重复元素，我们只需要找到其中的最小元素的位置。

最小元素位置是可以通过二分的方法获取到的：对于区间[beg, end]，由于数组a是逆时针旋转，因此a[end]<a[beg]成立。所以我们将a[mid]与a[end]比较，如果前者大，那么就移动beg，如果后者大，那么区间[mid,end]之间的数字已经是顺序排列了，紧接着缩小end，直到beg=end，说明找到了最小元素所在的位置。

得知了最小元素的位置，我们就可以在二分搜索的时候将旋转的因素考虑进去：对于某个位置i，(i-pivot) % len就是它原本所在的位置。因此我们可以将区间[beg, end]视为已拍好序数组a'中的元素下标，通过计算real_mid = (mid+pivot) % len得到其真正中间元素的位置，然后与target比较。最后返回匹配的real_mid位置即可。

int len = nums.size();
int beg = 0, end = len - 1, mid, realmid;
while (beg < end) { // find smallest element
    mid = (beg + end) >> 1;
    if (nums[mid] > nums[end]) beg = mid + 1;
    else end = mid; // 由于除法的截断性，mid<end一定成立，否则会退出循环，因此这一步一定会将end至少减1
}
int pivot = beg;
beg = 0, end = len - 1;
while (beg <= end) {
    mid = (beg + end) >> 1;
    realmid = (mid + pivot) % len;
    if (target < nums[realmid]) end = mid - 1;
    else if (target > nums[realmid]) beg = mid + 1;
    else return realmid;
}
return -1;

34. Find First and Last Position of Element in Sorted Array

medium 2019-07-23

二分查找的变体：给定数组nums和数字target，寻找nums中第一个不小于target的元素下标。

与寻找“等于target的元素”的一般二分查找不同，在寻找到等于target的元素时，仍然不能停止查找，由于要找的“最左”的元素，因此此时需要将end移动至该元素位置。直到beg==end，查找才终止，并返回。

通过这种方式我们就找到了某个元素的起始位置，还要找到结束位置。我们可以简单地寻找不小于target+1的元素下标，得到的就是“结束位置+1”。最后我们还需要检查找到的下标是否超出数组范围，以及对应元素是否为target。

int findStart(vector<int> &nums, int beg, int end, int target) {
    while (beg < end) {
        int mid = ((end - beg) >> 1) + beg; // 防止 beg + end 溢出
        if (nums[mid] >= target) end = mid;
        else beg = mid + 1;
    }
    return beg;
}
vector<int> searchRange(vector<int> &nums, int target) {
    int start = findStart(nums, 0, nums.size(), target); 
    // 注意 end 为数组末尾的后一个元素，由于 beg==end 时算法就终止，及除法的截断性，不会访问到 end 元素
    if (start == nums.size() || nums[start] != target) return {-1, -1};
    int end = findStart(nums, 0, nums.size(), target + 1);
    return {start, end - 1};
}

35. Search Insert Position

easy 2019-03-20

基本的二分查找。查找的是数组中“小于等于”target的最大的元素的下标。

int mid, beg = 0, end = nums.size(); // end指向数组后的位置
while (beg < end) {
    mid = (beg + end) >> 1;
    if (nums[mid] == target) return mid; // 一旦匹配就返回，可以提高一点效率，去掉也是可以的
    else if (nums[mid] < target) beg = mid + 1;
    else end = mid;
}
return beg;

当然也可以直接用lower_bound函数直接获取结果：

auto low = lower_bound(nums.begin(), nums.end(), target); // 返回指向“不小于”target的元素的迭代器
return low - nums.begin();

38. Count and Say

easy 2019-03-20

对于一个数，从高位向低位，连续有m个a，那么下一个数有2位就是ma，以此类推。

由于第i个数地生成仅依赖于第i-1个数，因此可以用递归的方式，先生成第i-1个数，然后再处理生成第i 个数。

if (n == 1) return "1";
string s = countAndSay(n - 1), res = "";
for (int i = 0; i < s.size(); i++) {
    int count = 1;
    while (i < s.size() - 1 && s[i] == s[i + 1])
        i++, count++;                    // 计算共有 count 个 s[i] 相邻
    res.append(to_string(count) + s[i]); // 将 count 和 s[i] 添加到下一个数中
}
return res;

39. Combination Sum

medium 2019-07-24

DFS。将数组每一个位置上的数，都计算0,1,⋯ ,⌊target/ai⌋0,1,\cdots,\lfloor target/a_i\rfloor0,1,⋯,⌊target/ai​⌋次，将数组遍历完后，就可以得到所有加和恰好为target的元素的组合，这种方法的复杂度为O(∏i=1n⌊targetai⌋)O(\prod_{i=1}^n\lfloor\frac{target}{a_i}\rfloor)O(∏i=1n​⌊ai​target​⌋)。由于要求的是元素之和为target，因此更小的元素更有可能被作为其中的一部分，所以我们先将数组排个序，然后在增加元素时一旦和超过target，就不再继续搜索。(如果不排序，那么即使当前及之前元素的和已经超过target，也无法断定之后没有更小的元素来取代当前的元素，从而满足条件)

void dfs(vector<int> &cand, int i, int target, vector<int> path, vector<vector<int>> &ans) {
    if (target == 0) { ans.push_back(path); return; }
    for (int j = i; j < cand.size() && cand[j] <= target; j++) {
        path.push_back(cand[j]);
        dfs(cand, j, target - cand[j], path, ans); // 将第j个元素加入，并继续寻找和为 target-a[j] 的其它元素
        path.pop_back();
    }
}
vector<vector<int>> combinationSum(vector<int> &candidates, int target) {
    vector<vector<int>> ans;
    sort(candidates.begin(), candidates.end());
    dfs(candidates, 0, target, vector<int>(), ans); // 传递空数组参数可以直接使用 vector<int>()
    return ans;
}

40. Combination Sum II

medium 2019-07-24

与上一题类似，只是数组中存在重复元素，且每个元素仅能使用一次。这时只需要将dfs()函数中递归调用的部分更改一下即可：

if (j == i || cand[j] != cand[j - 1])
		dfs(cand, j + 1, target - cand[j], path, ans);

直接通过a[j]!=a[j-1]的条件来过滤是否合理呢？是否会误过滤掉例如1,1,2这样的有效解呢？答案是否。我们在计算和的时候，每到一个节点都分了叉(选择 / 不选择)，因此最后得到的是一棵二叉树，当从根到节点的路径之和恰好为target时，就视为满足条件。因此这个过滤条件只是过滤了树中同一层的相同节点来避免重复，而对于类似1,1,2这种重复，两个1是处于树的不同层的，所以并不会被过滤掉。
