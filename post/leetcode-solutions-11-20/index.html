11. Container With Most Water

medium 2019-07-18

算法描述：2个指针i和j分别指向数组的首元素和尾元素。然后计算当前所围成的面积S=(j-i)*min(a[i],a[j])，然后将指向较小的元素的指针向内移动，并不断更新最大面积，直到2个指针相遇。

证明(反证法)：

假设我们得到的面积不是最大的面积，那么一定存在2个元素a[l]和a[r]构成了最大的面积。由于算法中2个指针相遇才会停止，因此必定碰到了a[l]和a[r]中的一个而没有碰到另一个。不失一般性，我们假设当前左边指针指向了a[l]，那么知道如下2种情况它才会移动：

  1. 2个指针撞到了一起，算法终止。那么由于另一个指针从右向左，一定会碰到a[r]，与假设矛盾。
  2. 右边指针先指向了元素a[r']>a[l]，从而使得左边指针右移，离开了a[l]，那么可以发现，a[l]与a[r']构成的面积一定大于a[l]与a[r]构成的面积(高度、宽度都更大)，与假设矛盾。

因此我们的算法一定能够获得最大的面积。

我们还可以从另一个角度考虑 链接：

指针先是指向首和尾，计算出了当前面积，假设a[i]<a[j]，那么a[i]与a[k],i<k<j的面积都更小，不用再计算。然后右移指针i,计算当前面积，假设a[i]>a[j]，那么a[k],i<k<j与a[j]的面积也更小，不用再计算。由于当i=j时算法结束，指针移动会移动n-1次，并且在所有的Cn2C_n^2Cn2​种组合中，我们一共省略了(n−2)+(n−1)+⋯+0=(n−2)(n−1)2(n-2)+(n-1)+\cdots+0=\frac{(n-2)(n-1)}{2}(n−2)+(n−1)+⋯+0=2(n−2)(n−1)​次面积的计算，剩下了Cn2−(n−2)(n−1)2=n−1C_n^2-\frac{(n-2)(n-1)}{2}=n-1Cn2​−2(n−2)(n−1)​=n−1种可能，一定能在n-1次计算中考虑到，因此我们得到的是最优解，且复杂度为O(n)O(n)O(n)。

12. Integer to Roman

medium 2019-07-18

每一个十进制位可以分为9, 5-8, 4, 0-3这四种情况来考虑进行处理，并且要结合该位在数中的位置。

unordered_map<int, char> m = {{1, 'I'}, {5, 'V'}, {10, 'X'}, {50, 'L'}, {100, 'C'}, {500, 'D'}, {1000, 'M'}};
int base = 1; string ans = "";
while (num) {
    int digit = num % 10; // 分情况处理还有可以合并的空间
    if (digit == 9) ans = ans + m[base * 10] + m[base];
    else if (digit >= 5) { for (int i = 5; i < digit; i++) { ans += m[base]; }  ans += m[base * 5]; }
    else if (digit == 4) ans = ans + m[base * 5] + m[base];
    else { for (int i = 0; i < digit; i++) ans += m[base]; }
    num /= 10; base *= 10;
}
reverse(ans.begin(), ans.end()); // 将字符串倒过来
return ans;

13. Roman to Integer

easy 2019-02-10

除了通过许多if和else语句逐个判断的naive方法外，考虑到输入的罗马数字“一定合法”，因此对于相邻的两个字符，根据前者和后者的大小关系，选择增或减前者的大小即可。

再使用unordered_map容器存储字符与数字的映射关系，通过Hash桶的存储方式，可以在O(1)O(1)O(1)时间内完成插入、删除和查找操作。

if (s.empty()) return 0;
unordered_map<char, int> m = {{'I', 1}, {'V', 5}, {'X', 10}, {'L', 50}, {'C', 100}, {'D', 500}, {'M', 1000}};
int num = m[s.back()];
for (int i = s.size() - 2; i >= 0; i--) 
    num += (m[s[i]] >= m[s[i + 1]] ? m[s[i]] : -m[s[i]]); // 如果前者小于后者，那么减去前者
return num;

14. Longest Common Prefix

easy 2019-02-10

将字符串对齐然后逐个比较即可。注意几点语法tips:

  1. for循环中遍历容器时，循环变量类型用 size_t 而非 int
  2. 对 string 进行截断可以用函数 resize(len)，表示将原字符串截断至长度len
  3. 记得处理边界情况：字符串数组为空
if (strs.empty()) return "";
string prefix;
for (size_t c_idx = 0; c_idx < strs[0].size(); prefix += strs[0][c_idx++])
    for (size_t str_idx = 1; str_idx < strs.size(); str_idx++)
        if (c_idx >= strs[str_idx].size() || strs[str_idx][c_idx] != strs[0][c_idx])
            return prefix;
return prefix;

15. 3Sum

medium 2019-07-18

之前的2Sum平凡的解法是遍历所有的二元组，复杂度为O(n2)O(n^2)O(n2)；先对数组排序，然后遍历数组并使用二分搜索的方法，复杂度为O(nlg⁡n)O(n\lg n)O(nlgn)；使用哈希表将每个元素作为键，复杂度可以降到O(n)O(n)O(n)。

而对于3Sum，如果遍历所有的三元组，复杂度为O(n3)O(n^3)O(n3)，显然是不尽人意的。我们希望能够找到复杂度更低的算法。

我们仍然采用先将数组排好序，然后固定一个元素的方法，然后使用2个指针i和j分别指向剩下元素的首和尾，并且判断三个数的和，如果大于0，那么将j左移；如果小于0,就将i右移；如果等于0，就将这个组合存入结果中，直到i=j迭代结束。由于题目要求结果不能出现重复，因此我们移动i和j直到与目前元素不同。同样，对于固定元素，在一轮迭代之后，也要更换为不同的元素。

使用这种方法，只需要2个循环，复杂度可以降到O(n2)O(n^2)O(n2)。

vector<vector<int>> res;
int len = nums.size();
if (len < 3) return res;
sort(nums.begin(), nums.end());
for (int i = 0; i < len - 2;) {
    int beg = i + 1, end = len - 1, pivot = nums[i];
    while (beg < end) { // 首尾指针
        int sum = pivot + nums[beg] + nums[end];
        if (sum < 0) beg++;
        else if (sum > 0) end--;
        else {
            res.push_back({pivot, nums[beg], nums[end]});
            int j = nums[beg], k = nums[end];
            while ((++beg) < len && nums[beg] == j) ;
            while ((--end) > i && nums[end] == k) ;
        }
    }
    while ((++i) < len - 2 && pivot == nums[i]) ;
}
return res;

16. 3Sum Closest

medium 2019-07-18

与上一题类似，由于题目说明了只拥有唯一解，因此就不需要之前麻烦地移动至下一个不同的元素(当然也可以移动来提高效率)。在每次迭代的过程中保存当前sum与target最小的差异值，如果有更接近的sum，那么更新差异值并保存当前的sum。最后返回最接近的sum即可。复杂度也是O(n2)O(n^2)O(n2)。

17. Letter Combinations of a Phone Number

medium 2019-07-19

典型的字符串回溯，用DFS来解。注意处理空串的情况。附上代码：

vector<string> letterCombinations(string digits) {
    vector<string> letter({"abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"});
    vector<string> ans;
    if (digits.size() == 0) return ans;
    dfs(digits, 0, "", letter, ans); // 按照当前的digit和所在位置进行DFS搜索
    return ans;
}
void dfs(string &digits, int i, string cur, vector<string> &letter, vector<string> &ans) {
    if (i == digits.size()) { ans.push_back(cur); return; }
    int idx = digits[i] - '2';
    for (int j = 0; j < letter[idx].size(); j++) { dfs(digits, i + 1, cur + letter[idx][j], letter, ans); }
}

19. Remove Nth Node From End of List

medium 2019-07-18

快慢指针。快指针先于慢指针N步，当快指针到达链表尾时，删除慢指针所指向的元素。注意处理删除表头元素时，返回的表头会后移；以及从内存中释放被删除元素占用的空间。

ListNode *p = head, *q = head, *r = head;
while (n--) q = q->next;
if (!q) return p->next; // 删除的是表头元素
while (q->next) { p = p->next; q = q->next; }
// ListNode *del = p->next;
p->next = p->next->next;
// delete del;
return r;
