17. 电话号码的字母组合

medium 2019-07-19

给定一个仅包含数字 2-9 的字符串，返回所有它能表示的字母组合。数字与字母的映射与电话按键相同。

思路一：DFS

DFS典型的字符串回溯，用DFS来解。注意处理空串的情况。附上代码：

vector<string> letterCombinations(string digits) {
    vector<string> letter({"abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"});
    vector<string> ans;
    if (digits.size() == 0) return ans;
    dfs(digits, 0, "", letter, ans); // 按照当前的digit和所在位置进行DFS搜索
    return ans;
}
void dfs(string &digits, int i, string cur, vector<string> &letter, vector<string> &ans) {
    if (i == digits.size()) { ans.push_back(cur); return; }
    int idx = digits[i] - '2';
    for (int j = 0; j < letter[idx].size(); j++) { dfs(digits, i + 1, cur + letter[idx][j], letter, ans); }
}

22. 括号生成

medium 2019-07-20

给出 n 代表生成括号的对数，请你写出一个函数，使其能够生成所有可能的并且有效的括号组合。

思路一：DFS

字符串回溯，使用DFS即可。注意已生成的字符串中，任意前k个字符，左括号的数目都不小于右括号。

vector<string> generateParenthesis(int n) {
    vector<string> ans;
    dfs(n, n, "", ans);
    return ans;
}
void dfs(int left, int right, string cur, vector<string> &ans) {
    if (right == 0) { ans.push_back(cur); return; }
    if (left < right) dfs(left, right - 1, cur + ')', ans); // 剩余右括号一定不能小于左括号
    if (left > 0) dfs(left - 1, right, cur + '(', ans); // 左括号消耗完之前都可以继续加
}

思路二：闭合形式 - 迭代

考虑到n对有效括号的排列数目是卡特兰数Cn=C2nn/(n+1)C_n=C_{2n}^{n}/(n+1)Cn​=C2nn​/(n+1)，而卡特兰数有递推公式：Cn+1=∑i=0nCiCn−i with C0=1C_{n+1}=\sum_{i=0}^nC_iC_{n-i}\text{ with }C_0=1Cn+1​=∑i=0n​Ci​Cn−i​ with C0​=1。具体到括号对排列问题上，可以表达为组合( left ) right的形式，也就是固定1对括号，然后变动剩下的n-1对括号的位置，而剩下的n-1对括号又有n-1种组合。所以我们可以先计算出较小的n的组合数，然后再生成n对括号的排列。

vector<vector<string>> par(n+1);
par[0].push_back("");
for(int i=1;i<=n;i++) {
    for(int j=0;j<i;j++) {
        for(int p=0;p<par[j].size();p++) {
            for(int q=0;q<par[i-j-1].size();q++) {
                par[i].push_back("("+ par[j][p] + ")" + par[i-j-1][q]);
            }
        }
    }
}
return par[n];

37. 解数独

hard 2020-02-22

给定一个9×9的棋盘，按照数独的规则填充空格。一个典型的DFS问题。但是可以通过已有的数字来减少回溯时的可选项。即使用三个二维数组row,col,box，例如row[i][no]=0/1表示第i行是否还能够填充数字no。

首先用棋盘中已经存在的数字去更新这三个数组，并且之后在搜索的时候每填充一个数字，就更新这三个数组，使得后续搜索的可选项减少。DFS部分的实现如下所示：

// board为字符类型的棋盘，pos为可以放置数字的坐标数组，并对其中第p个坐标放置
bool dfs(vector<vector<char>> &board, vector<pair<int, int>> &pos, int p) {
    if (p == pos.size()) // 如果所有位置都已经放上数字，那么说明得到一个可行解
        return true;
    int i = pos[p].first, j = pos[p].second;
    for (int k = 1; k <= 9; k++) {
        // 遍历所有可以放置的数字，并使用row,col,box三个数组来排除已经存在的数字
        if (row[i][k] == 0 && col[j][k] == 0 && box[BOX(i, j)][k] == 0) {
            board[i][j] = k + '0';
            mark(i, j, k, 1);
            if (dfs(board, pos, p + 1)) return true; // 如果找到可行解，立刻返回
            mark(i, j, k, 0); // 回溯，尝试下一个数字
        }
    }
    return false;
}

需要注意的是，在找到可行解的时候，要立刻返回，否则就会被认为需要回溯，并填充上其它的数字。

39. 组合总和

medium 2019-07-24

DFS。将数组每一个位置上的数，都计算0,1,⋯ ,⌊target/ai⌋0,1,\cdots,\lfloor target/a_i\rfloor0,1,⋯,⌊target/ai​⌋次，将数组遍历完后，就可以得到所有加和恰好为target的元素的组合，这种方法的复杂度为O(∏i=1n⌊targetai⌋)O(\prod_{i=1}^n\lfloor\frac{target}{a_i}\rfloor)O(∏i=1n​⌊ai​target​⌋)。由于要求的是元素之和为target，因此更小的元素更有可能被作为其中的一部分，所以我们先将数组排个序，然后在增加元素时一旦和超过target，就不再继续搜索。(如果不排序，那么即使当前及之前元素的和已经超过target，也无法断定之后没有更小的元素来取代当前的元素，从而满足条件)

void dfs(vector<int> &cand, int i, int target, vector<int> path, vector<vector<int>> &ans) {
    if (target == 0) { ans.push_back(path); return; }
    for (int j = i; j < cand.size() && cand[j] <= target; j++) {
        path.push_back(cand[j]);
        dfs(cand, j, target - cand[j], path, ans); // 将第j个元素加入，并继续寻找和为 target-a[j] 的其它元素
        path.pop_back();
    }
}
vector<vector<int>> combinationSum(vector<int> &candidates, int target) {
    vector<vector<int>> ans;
    sort(candidates.begin(), candidates.end());
    dfs(candidates, 0, target, vector<int>(), ans); // 传递空数组参数可以直接使用 vector<int>()
    return ans;
}

40. 组合总和 II

medium 2019-07-24

与上一题类似，只是数组中存在重复元素，且每个元素仅能使用一次。这时只需要将dfs()函数中递归调用的部分更改一下即可：

if (j == i || cand[j] != cand[j - 1])
		dfs(cand, j + 1, target - cand[j], path, ans);     
// 或者
if(j > i && cand[j] == cand[i]) continue;
dfs(cand, j + 1, target - cand[j], path, ans);

直接通过a[j]!=a[j-1]的条件来过滤是否合理呢？是否会误过滤掉例如1,1,2这样的有效解呢？答案是否。我们在计算和的时候，每到一个节点都分了叉(选择 / 不选择)，因此最后得到的是一棵二叉树，当从根到节点的路径之和恰好为target时，就视为满足条件。因此加上过滤条件i==j就是过滤了树中同一层的相同节点来避免重复，而对于类似1,1,2这种重复，两个1是处于树的不同层的，有j>i，所以并不会被过滤掉。

46. 全排列

medium 2019-07-24

方法一：与31题类似，只需要先计算出一共有n!n!n!种排列，然后将next_permutation()函数执行n!n!n!次就可以了。

方法二：逐个插入法。

例如对于序列[1,2,3]，刚开始只有1；接着数字2有两种插入方式，构成[1,2]和[2,1]；对于这两个序列，分别有3个插入位置，对于序列[1,2]可以构成[3,1,2], [1,3,2]和[1,2,3]，另一个也类似。

对于长度为nnn的序列，通过这种方式也可以直观地得知有n!n!n!种排列方法。在实现的过程中，可以通过DFS来实现。

// caller: dfs(nums, 0, vector<int>(), ans)
void dfs(vector<int> &nums, int pos, vector<int> perm, vector<vector<int>> &ans) {
    if (pos == nums.size()) { ans.push_back(perm); return; }
    for (int i = 0; i <= perm.size(); i++) { // 对于每一个当前的序列，都在每个空隙中插入新的数
        perm.insert(perm.begin() + i, nums[pos]);
        dfs(nums, pos + 1, perm, ans);
        perm.erase(perm.begin() + i);
    }
}

方法三：从前到后依次固定每个数。

例如对于序列[1,2,3]，如果先固定1，那么就是[2,3]两个数的排列；之后再固定2，剩下[1,3]两个数的排列；最后再固定3，剩下[1,2]两个数的排列。

实现过程中仍然使用DFS的方式，通过swap方法依次将每个数字固定在最前，然后递归调用之后的序列。

// caller: dfs(nums, 0, ans)
void dfs(vector<int> &nums, int pos, vector<vector<int>> &ans) {
    if (pos == nums.size() - 1) { ans.push_back(nums); return; } // 当固定了最后一个数时，就返回
    for (int i = pos; i < nums.size(); i++) { // 依次将每个数固定在最前
        swap(nums[pos], nums[i]);
        dfs(nums, pos + 1, ans);
        swap(nums[pos], nums[i]);
    }
}

方法四：设立一个visited数组，直接DFS即可

void dfs(int i, vector<int> &nums, vector<bool> &vis, vector<int> &cand, vector<vector<int>> &ans) {
    if (i == nums.size()) { ans.push_back(cand); return; }
    for (int j = 0; j < nums.size(); j++) {
        if (vis[j] == false) {
            cand[i] = nums[j];
            vis[j]  = true;
            dfs(i + 1, nums, vis, cand, ans);
            vis[j] = false;
        }
    }
}

47. 全排列 II

medium 2019-09-13

方法一：类似上一题的方法三，只不过每次固定一个数时，如果已经出现过，那么就跳过。例如序列[1,1,2]，先固定了1，剩下[1,2]两个数的排列；接下来就不用固定第二个1了，只需要固定2，剩下[1,1]两个数的排列。

实现时可以使用set或map的结构来确定之前是否已经固定过某个数，如果固定过就直接跳过即可。

方法二：方法一的变体。我们发现，如果事先将数组排个序，那么相同的数字变成相邻，判断是否重复的时候，只需要顺次遍历序列，然后在数字变为不同时再进行交换，就可以保证不重复。实现代码如下：

// caller: dfs(nums, 0, ans)
void dfs(vector<int> nums, int pos, vector<vector<int>> &ans)  // nums传值
    if (pos == nums.size() - 1) { ans.push_back(nums); return; }
    for (int i = pos; i < nums.size(); i++) {
        if (i == pos || nums[i] != nums[pos]) {
            swap(nums[pos], nums[i]);
            dfs(nums, pos + 1, ans);
        }
    }
}

有两点需要注意，一是当i==pos时，表示固定的是第一个元素，需要考虑进去。二是传入的nums数组是传值，如果是传引用，对于上一题，不存在重复的判断，这种方法依旧会将所有的可能性输出，不会出错；而对于这一题，所有dfs()函数使用的都是同一个数组，判断的时候可能由于nums[i]的值已经发生了变化，而导致if语句判断出错，所以我们只能使用传值这种空间效率比较低的方式。

方法三：直接DFS

void dfs(int i, vector<int> &nums, vector<bool> &vis, vector<int> &cand, vector<vector<int>> &ans) {
    if (i == nums.size()) { ans.push_back(cand); return; }
    for (int j = 0; j < nums.size(); j++) {
        if (vis[j] == false) {
            // 前两个条件是过滤重复元素，最后一个条件是避免不同位置的相同元素的覆盖
            // 如果vis[j-1]=true,说明当前是在不同的i(下一个位置)搜索，不需要过滤相同元素
            if (j != 0 && nums[j - 1] == nums[j] && vis[j - 1] == false)
                continue;
            cand[i] = nums[j];
            vis[j]  = true;
            dfs(i + 1, nums, vis, cand, ans);
            vis[j] = false;
        }
    }
}

51. N皇后

hard 2019-10-07

借鉴52题的基于位运算方法的实现。

bool notin(int j, vector<int> &a) {
    for (auto i : a) if (i == j) return false;
    return true;
}

void add(vector<int> &queue, int n, vector<vector<string>> &ans) {
    vector<string> board(n, string(n, '.')); // 初始化所有的 string 为 '...'
    for (int i = 0; i < n; i++) 
        board[i].replace(queue[i], 1, 1, 'Q');
    ans.push_back(board);
}
// caller:  vector<int> queue, diff, sum;   dfs(queue, diff, sum, 0, n, ans);
void dfs(vector<int> &queue, vector<int> &diff, vector<int> &sum, int i, int n, vector<vector<string>> &ans) {
    int row = queue.size();
    if (row == n) {
        add(queue, n, ans);
        return;
    }
    for (int j = 0; j < n; j++) {
        if (notin(j, queue) && notin(row - j, diff) && notin(row + j, sum)) {
            queue.push_back(j); diff.push_back(row - j); sum.push_back(row + j);
            dfs(queue, diff, sum, i + 1, n, ans);
            queue.pop_back(); diff.pop_back(); sum.pop_back();
        }
    }
}

首先建立三个空数组queue, diff, sum；queue存储了第i行所放置的皇后的列，diff存储了皇后所在的行数与列数之差，用来避免同一对角线，sum存储了皇后所在的行数与列数之和，用来避免同一反对角线。对每一列遍历时，都先检查所在的列是否在这三个数组中，如果都不在，那么说明该位置可以放置，在这三个数组中添加该位置，并搜寻下一行。

当queue中所有n列都有皇后之后，就得到一个可行解，根据queue来画出棋盘的布局，并加入最终结果中。

52. N皇后 II

hard 2019-10-05

典型的搜索问题，套用DFS的套路即可。

typedef pair<int, int> point;

bool check(vector<point> &queen, int row, int col) {
    for (int i = 0; i < queen.size(); i++) {
        point p = queen[i];
        if (p.first == row || p.second == col) return false;
        if (row - col == p.first - p.second) return false;
        if (row + col == p.first + p.second) return false;
    }
    return true;
}

// queen: 存放皇后放置的位置；n:棋盘边长；i:当前准备放置的行数；ans:放置种类的统计
void dfs(vector<point> &queen, int n, int i, int &ans) {
    for (int j = 0; j < n; j++) { // 对于第i行每一列j都要尝试
        if (check(queen, i, j)) { // 如果当前位置可以放置
            if (i == n - 1) ans++; // 如果已经是最后一行，那么得到了一种可行方案
            else {
                queen.push_back(make_pair(i, j)); // 否则将该位置放入queen中
                dfs(queen, n, i + 1, ans); // 尝试下一行
                queen.pop_back(); // 在第i行换个位置继续尝试
            }
        }
    }
}

这种方法算是一种穷举的方法，时间复杂度是O(nn)O(n^n)O(nn)。实际上，N皇后问题是NP问题，没有多项式时间的解法。有一种位运算的解法，虽然理论时间复杂度没有变化，但是其操作对于CPU来说更加方便，所以耗时要比一般的算法要少一些：

// caller: dfs(0, 0, 0, ans, (1 << n) - 1);    (ans初始化为0)
void dfs(int &&row, int &&ld, int &&rd, int &ans, const int limit) {
    if (row != limit) {
        int pos = limit & (~(row | ld | rd));
        while (pos) {
            int p = pos & (~pos + 1);
            pos -= p;
            dfs(row | p, (ld | p) << 1, (rd | p) >> 1, ans, limit);
        }
    } else {
        ans++;
    }
}

首先，所有int型变量都看成bit set，其中limit=(1<<n)-1，是n位全为1的掩码。程序一行一行地寻找可以放置的位置，三个变量col, ld, rd中的1表示这一行在 列、反对角线、对角线 的限制下，哪些列不可以放置。

程序先判断，在列的限制下，是否所有的列都不可以放置，如果是，说明所有的列都有一个皇后，那么说明找到了一个可行解；如果不是，那么通过计算limit & (~(row | ld |rd))找到该行还剩下哪些列可以放置，如果没有列可以放置(pos=0)，那么说明该尝试无效，于是进行回溯；如果有列可放，那么通过pos & (~pos+1)找到最低位的1，在它所在的位表示的列放入一个皇后，从pos中将这个1减去表示这列已经放过皇后，并更新row |= p，ld = (ld | p) << 1，rd = (rd | p) >> 1，表示在下一行这些列是不可以放置的，接着搜索下一行。下图是一个示例。

这种算法虽然没有达到更优的渐进复杂度，但是可以这样考虑：对于每一行，一旦确定了皇后的位置，那么下一行的至少2个位置是不用再考虑的，即T(n)≤(n−2)T(n−1)+O(1)T(n)\le (n-2)T(n-1)+O(1)T(n)≤(n−2)T(n−1)+O(1)，则T(n)=O((n−2)(n−4)⋯ )T(n)=O((n-2)(n-4)\cdots)T(n)=O((n−2)(n−4)⋯)，耗时还是少了许多。

60. 第k个排列

给出集合 [1,2,3,…,n]，其所有元素共有n!n!n!种排列。按大小顺序列出所有排列情况，并给出第k个排列。

思路一：普通DFS

使用DFS的策略，从最小的元素开始，逐个地生成每个数字，直到第k个数字。

void permute(int n, int &k, vector<char> &num, vector<bool> &used) {
    if (n == 0) k--;
    else {
        for (int i = 1; i < num.size(); i++) {
            if (used[i] == false) {
                num[n]  = (i + '0');
                used[i] = true;
                permute(n - 1, k, num, used);
                if (k == 0) return;
                used[i] = false;
            }
        }
    }
    return;
}

string getPermutation(int n, int k) {
    vector<char> num(n + 1, 0);
    vector<bool> used(n + 1, false);
    permute(n, k, num, used);
    return string(num.rbegin(), num.rend());
}

思路二：DFS + 剪枝

对于一个n位的数，那么如果前i位固定，后面的数字的可能取值数目为(n-i)!，利用这一点，在DFS的搜索过程中，如果固定第前i位为x，就可以通过计算后面数字的取值数目与k相比，来判断以x为前缀的排列是否会产生出第k个排列。

如果为否，那么以x为前缀的排列就不需要再考虑了，否则第k位的前缀一定为x，接着再考虑后面的位。

string ans;
// i: 已经找到了前i位的数字
void dfs(int i, int n, int k, vector<bool> &used, vector<int> &fac) {
    if (i == n)
        return;
    int cnt = fac[n - 1 - i];
    for (int j = 1; j <= n; j++) {
        if (!used[j]) {
            if (cnt < k) { // 查看在固定第i+1位后，k能否落在(n-1-i)!范围内
                k -= cnt;
            } else { // 如果k在(n-1-i)!范围内，那么就固定了第i+1位为j，继续搜索后面的位
                ans.push_back(j + '0');
                used[j] = true;
                dfs(i + 1, n, k, used, fac);
            }
        }
    }
}

string getPermutation(int n, int k) {
    vector<int> fac(n, 1);
    for (int i = 1; i < n; i++) fac[i] = fac[i - 1] * i;
    vector<bool> used(n + 1, false);
    dfs(0, n, k, used, fac);
    return ans;
}

思路三：数学 - 康托展开

vector<int> fac(n, 1);
vector<bool> used(n + 1, false);
for (int i = 1; i < n; i++) fac[i] = fac[i - 1] * i;

    int j = k / fac[n - i]; // 当前要使用的数字前面有j个“未使用过的”数字
    int p = 1, q = j;
    while (q) { // 过滤掉j个“未使用过的”数字
        if (!used[p++])
            q--;
    }
    while (used[p]) p++; // 过滤掉“使用过的”数字
    ans.push_back(p + '0');
    used[p] = true;
    k -= j * fac[n - i];
}

78. 子集

给定一组不含重复元素的整数数组 nums，返回该数组所有可能的子集（幂集）。

对于某个数组的子集，其中的每个元素都可以选择加入或不加入，依次进行DFS即可。

vector<vector<int>> ans;
void dfs(int i, int n, vector<int> &nums, vector<int> &tmp) {
    if (i == n) {
        ans.push_back(tmp);
        return;
    }
    dfs(i + 1, n, nums, tmp);
    tmp.push_back(nums[i]);
    dfs(i + 1, n, nums, tmp);
    tmp.pop_back();
}

vector<vector<int>> subsets(vector<int> &nums) {
    int n = nums.size();
    vector<int> tmp;
    dfs(0, n, nums, tmp);
    return ans;
}

或者也可以使用将数组中的某个位置使用二进制中的一个位来表示，那么就是遍历00...0到11...1之间所有的数，将相应位置的元素加入到结果数组中即可，复杂度是一样的，为O(N×2N)O(N\times 2^N)O(N×2N)。

130. 被围绕的区域

给定一个二维的矩阵，包含 'X' 和 'O'。找到所有被 'X' 围绕的区域，并将这些区域里所有的 'O' 用 'X' 填充。

思路一：DFS

从所有边缘的'O'点开始向四周所有可达的'O'进行DFS，并将这些位置均做标记。在将所有的'O'遍历完成后，剩余没有标记的位置都可以填充为'X'。

int dir[4][2] = {{1, 0}, {0, 1}, {-1, 0}, {0, -1}};
void dfs(vector<vector<char>> &board, vector<vector<bool>> &reg, int i, int j, int m, int n) {
    reg[i][j] = true;
    for (int k = 0; k < 4; k++) {
        int ni = i + dir[k][0], nj = j + dir[k][1];
        if (0 <= ni && ni < m && 0 <= nj && nj < n && board[ni][nj] == 'O' && reg[ni][nj] == false) {
            dfs(board, reg, ni, nj, m, n);
        }
    }
}
void solve(vector<vector<char>> &board) {
    int m = board.size();
    if (m == 0)
        return;
    int n = board[0].size();
    vector<vector<bool>> reg(m, vector<bool>(n, false));
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (board[i][j] == 'O' && reg[i][j] == false) {
                dfs(board, reg, i, j, m, n);
            }
            if (i != 0 && i != m - 1) // 过滤到矩形的非边缘结点
                j += (n - 2);
        }
    }
    for (int i = 1; i < m - 1; i++) {
        for (int j = 1; j < n - 1; j++) {
            if (board[i][j] == 'O' && reg[i][j] == false) {
                board[i][j] = 'X';
            }
        }
    }
}

思路二：并查集

并查集经常用来解决区域连通性的问题，在这里可以使用并查集来合并所有的连通区域，另外将所有边缘的'O'点都与一个虚拟的结点合并，最后不与虚拟节点在一个连通区域的'O'都可以变为'X'。

vector<int> parent;
vector<int> rank;
int M, N;

void makeset() {
    parent.resize(M * N + 1);
    iota(parent.begin(), parent.end(), 0);
    rank.resize(M * N + 1);
    fill(rank.begin(), rank.end(), 0);
}

int find(int x) {
    if (parent[x] != x) parent[x] = find(parent[x]);
    return parent[x];
}

void Union(int x, int y) {
    int px = find(x), py = find(y);
    if (rank[px] < rank[py]) {
        parent[px] = py;
    } else {
        parent[py] = px;
        if (rank[px] == rank[py]) rank[px]++;
    }
}
inline int node(int i, int j) { return i * N + j; } // 将二维坐标变为一维，方便实现并查集
void solve(vector<vector<char>> &board) {
    M = board.size();
    if (M == 0) return;
    N = board[0].size();
    int dummy = M * N;
    makeset();
    for (int i = 0; i < M; i++) {
        for (int j = 0; j < N; j++) {
            if (board[i][j] == 'O') {
                if (i == 0 || j == 0 || i == M - 1 || j == N - 1) {
                    Union(node(i, j), dummy);
                } else {
                    if (i > 0 && board[i - 1][j] == 'O') {
                        Union(node(i, j), node(i - 1, j));
                    }
                    if (j > 0 && board[i][j - 1] == 'O') {
                        Union(node(i, j), node(i, j - 1));
                    }
                    if (i < M - 1 && board[i + 1][j] == 'O') {
                        Union(node(i, j), node(i + 1, j));
                    }
                    if (j < N - 1 && board[i][j + 1] == 'O') {
                        Union(node(i, j), node(i, j + 1));
                    }
                }
            }
        }
    }
    for (int i = 0; i < M; i++) {
        for (int j = 0; j < N; j++) {
            if (find(node(i, j)) != find(dummy))
                board[i][j] = 'X';
        }
    }
}

365. 水壶问题

有两个容量分别为x升和y升的水壶以及无限多的水。请判断能否通过使用这两个水壶，从而可以得到恰好z升的水？允许的操作只有3种：装满一个水壶，清空一个水壶，从一个水壶向另一个倒水，直到装满或倒空。

思路一：DFS

这个问题本身是一个搜索问题，搜索的状态选择集合为：{把x装满，把y装满，把x清空，把y清空，把x的水倒入y直到x为空或y为满，把y的水倒入x直到y为空或x为满}。

因此可以使用DFS的思想，但是一方面递归调用的层数可能过多，因此用栈来替代；另一方面在搜索过程中会有很多重复状态，因此需要用一个HashSet来规避重复搜索。

using P = pair<int,int>;
stack<P> s;
auto hash_function = [](const P &o) { return hash<int>()(o.first) ^ hash<int>()(o.second); };
// P类型没有默认的hash函数，因此需要自定义
unordered_set<P, decltype(hash_function)> m(0, hash_function);
s.emplace(0, 0);
while (!s.empty()) {
    if (m.count(s.top())) {
        s.pop();
        continue;
    }
    m.emplace(s.top());
    auto [remain_x, remain_y] = s.top();
    if (remain_x == z || remain_y == z || remain_x + remain_y == z) {
        return true;
    }
    s.emplace(remain_x, y);
    s.emplace(x, remain_y);
    s.emplace(remain_x, 0);
    s.emplace(0, remain_y);
    s.emplace(remain_x - min(remain_x, y - remain_y), remain_y + min(remain_x, y - remain_y));
    s.emplace(remain_x + min(x - remain_x, remain_y), remain_y - min(x - remain_x, remain_y));
}
return false;

思路二：数学

命题：每次操作只会给水的总量带来 x 或者 y 的变化量。

证明：由于题目中限制的操作，在任一时刻，至少有一个壶是空的或满的。而对一个不满的桶加水是没有意义的，因为这等价于将该桶直接加满水，而另一个桶要么是空，要么是满；对一个不空的桶倒水也是没有意义的，因为这等价于该桶回到初始状态，而另一个桶要么空，要么满。因此每次操作水的总量总是会变化x或y。

那么问题就变成了，是否存在整数a,b使得ax+by=z，其中z≤x+y。根据裴蜀定理，这等价于问z是否是x,y最大公约数的倍数。
