202. Happy Number

easy 2019-07-09

参考Wiki。对于m(m>3)位的数，在进行一次平方加和后，所得到的数最大为 81m；即所有超过1000的数，在若干次平方加和后都会小于1000。而小于1000的最大数为999，接下来发现所有3位数都会变成至多2位数。而对于任意小于100的数，经过逐个计算，要么它收敛到1，要么进入4→⋯→44\rightarrow \cdots \rightarrow 44→⋯→4的循环，因此可以通过这个条件进行判断。

while (n != 1 && n != 4) {
    int res = 0;
    while (n) { res += (n % 10) * (n % 10); n /= 10; }
    n = res;
}
return n == 1 ? true : false;

当然，如果不使用这个性质，还可以使用一个Hash表保存访问过的数，如果出现了循环，且并不为1，那么就返回false。

203. Remove Linked List Elements

easy 2019-07-03

对于该题而言，链表中的元素分三类：开头，中间，结尾。其中开头元素要单独处理，用以移动head的位置，另两类可以一并处理，但要注意边界情况。

while(head && head->val==val) head=head->next;
ListNode *p=head;
while(p && p->next) {
    if(p->next->val==val) p->next=p->next->next;
    else p=p->next;
}
return head;

204. Count Primes

easy 2019-08-26

埃氏筛法：首先假设所有数都是素数，从2开始到n\sqrt nn​，每个数的所有倍数(除自身外)都不是素数。最后统计素数的总数。时间复杂度为O(nlog⁡log⁡n)O(n\log \log n)O(nloglogn)。

vector<bool> prime(n + 1, true);
prime[0] = prime[1] = prime[n] = false; // 0和n不考虑，1不是素数
for (int i = 2; i < sqrt(n); i++) {
  	// 从 j = i^2 开始, 因为i的更小倍数已经被检测过了
    if (prime[i]) { for (int j = i * i; j < n; j += i)  prime[j] = false; }
}
return count(prime.begin(), prime.end(), true);

欧拉筛法：保证每个数只被测试一遍

206. Reverse Linked List

easy 2019-07-04

非递归的方式在检测链表是否有环的第#141题中已经实现，这里使用递归的方法，效果相同，写起来更简洁。

ListNode *reverseList(ListNode *p, ListNode *q) { // 初始调用时，p=head, q=NULL
    if (!p)  return q;
    ListNode *k = p->next;  p->next = q;
    return reverseList(k, p);
}

226. Invert Binary Tree

easy 2019-07-08

递归反转子树即可。

if (!root) return root;
swap(root->left, root->right); 
invertTree(root->left);
invertTree(root->right);
return root;

231. Power of Two

easy 2019-07-04

只需要判断二进制形式是否满足10..0的模式，0要单独处理。

if (!n) return false;
while (n) {
    if ((n & 1) && (n ^ 1)) return false; // 如果n的末尾为1且n!=1,那么说明不满足要求
    n >>= 1;
}
return true;
